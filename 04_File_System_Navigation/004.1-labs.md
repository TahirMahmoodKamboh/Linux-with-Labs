# Module 4: Labs – File System Navigation

## Lab 1: Explore the Root Directory
- List the contents of the root directory.
  ```bash
  $ ls /
  bin  boot  dev  etc  home  lib  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var
  ```

## Lab 2: Navigate to Common Directories
- Go to your home directory, then to /etc, and back.
  ```bash
  $ cd ~
  $ pwd
  /home/student
  $ cd /etc
  $ pwd
  /etc
  $ cd -
  /home/student
  ```

## Lab 3: Practice Absolute and Relative Paths
- Move between directories using both path types.
  ```bash
  $ cd /home/student/Documents
  $ pwd
  /home/student/Documents
  $ cd ..
  $ pwd
  /home/student
  $ cd ./Documents
  $ pwd
  /home/student/Documents
  ```

## Lab 4: List Files with Options
- Try different `ls` flags.
  ```bash
  $ ls -lh
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  drwxr-xr-x 2 student student 4.0K Jan  4 09:59 docs
  $ ls -la
  drwxr-xr-x 2 student student 4096 Jan  4 09:59 .
  drwxr-xr-x 3 student student 4096 Jan  4 09:58 ..
  -rw-r--r-- 1 student student    0 Jan  4 10:00 .hiddenfile
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  $ ls -lt
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  drwxr-xr-x 2 student student 4.0K Jan  4 09:59 docs
  $ ls -d */
  docs/  downloads/
  ```

## Lab 5: Use tree, realpath, basename, dirname
- Display directory structure and manipulate paths.
  ```bash
  $ tree
  .
  ├── docs
  │   └── notes.txt
  └── file.txt
  $ realpath file.txt
  /home/student/file.txt
  $ basename /home/student/file.txt
  file.txt
  $ dirname /home/student/file.txt
  /home/student
  ```

## Solutions

### Lab 1 Solution
- The command `ls /` lists the contents of the root directory. You should see system folders like:
  ```bash
  bin  boot  dev  etc  home  lib  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var
  ```

### Lab 2 Solution
- Navigating to your home directory, then to /etc, and back:
  ```bash
  $ cd ~
  $ pwd
  /home/student
  $ cd /etc
  $ pwd
  /etc
  $ cd -
  /home/student
  ```
  - `cd -` returns you to your previous directory.

### Lab 3 Solution
- Using absolute and relative paths:
  ```bash
  $ cd /home/student/Documents
  $ pwd
  /home/student/Documents
  $ cd ..
  $ pwd
  /home/student
  $ cd ./Documents
  $ pwd
  /home/student/Documents
  ```
  - `cd ..` moves up one level; `./Documents` is a relative path from the current directory.

### Lab 4 Solution
- Trying different `ls` flags:
  ```bash
  $ ls -lh
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  drwxr-xr-x 2 student student 4.0K Jan  4 09:59 docs
  $ ls -la
  drwxr-xr-x 2 student student 4096 Jan  4 09:59 .
  drwxr-xr-x 3 student student 4096 Jan  4 09:58 ..
  -rw-r--r-- 1 student student    0 Jan  4 10:00 .hiddenfile
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  $ ls -lt
  -rw-r--r-- 1 student student 1.1K Jan  4 10:00 file.txt
  drwxr-xr-x 2 student student 4.0K Jan  4 09:59 docs
  $ ls -d */
  docs/  downloads/
  ```
  - `ls -lh` shows human-readable sizes, `ls -la` shows all files (including hidden), `ls -lt` sorts by modification time, and `ls -d */` lists only directories.

### Lab 5 Solution
- Using tree, realpath, basename, and dirname:
  ```bash
  $ tree
  .
  ├── docs
  │   └── notes.txt
  └── file.txt
  $ realpath file.txt
  /home/student/file.txt
  $ basename /home/student/file.txt
  file.txt
  $ dirname /home/student/file.txt
  /home/student
  ```
  - `tree` shows the directory structure, `realpath` gives the absolute path, `basename` extracts the filename, and `dirname` extracts the directory path.
